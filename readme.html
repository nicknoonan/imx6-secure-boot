U-boot with HAB build procedure

Environment setup

  1. Download the required source code using the git utility.
xxxxxxxxxx
$ git clone https://github.com/boundarydevices/u-boot-imx6 -b boundary-v2016.03
$ cd u-boot-imx6
  1. Initialize the U-boot building environment.
xxxxxxxxxx
$ export ARCH=arm
$ export CROSS_COMPILE=arm-linux-gnueabihf-
$ make nitrogen6q_defconfig
$ make menuconfig
  1. Enable i.MX HAB features inside the configuration cli gui

Building procedure

  1. Use make to start the build. Pipe the last 6 lines of output to a text file to take note of the load address, entry point, and HAB blocks.
xxxxxxxxxx
$ make V=1 -j8 > build.log && cat build.log | tail -n 6 > hab_blocks.txt
$ cat hab_blocks.txt
xxxxxxxxxx

Fuse values

  1. Generate the fuse values to be flashed on the board later using the srktool (super root key creation tool) included in the NXP code signing package. Fuse values will sequentially be stored in fusevals.txt.

$ cp u-boot_signed.imx u-boot.nitrogen6q

Signing the linux kernel

  1. Copy over zImage to the CST directory
x
$ cd ~/CST/linux64/bin
$ cp ~/imx6yocto/build/tmp/deploy/images/nitrogen6x/zImage .

Padding the zImage

  1. Determine the size of the zImage binary to be signed. Make note of its size and round to the nearest 4kB boundary. For example, 0x68e570 will round to 0x68f000
xxxxxxxxxx
$ hexdump zImage | tail -n 1

 

xxxxxxxxxx
068e570
  1. Pad zImage to the nearest 4kB boundary.
x
$ objcopy -I binary -O binary --pad-to=0x68f000 --gap-fill=0x00 zImage zImage-pad.bin

Create the IVT

  1. Create a script that will generate an IVT to be appended to the zImage.
xxxxxxxxxx
$ vim genIVT
xxxxxxxxxx
#! /usr/bin/perl -w
use strict;
open(my $out, '>:raw', 'ivt.bin') or die "Unable to open: $!";
print $out pack("V", 0x412000D1); # IVT Header
print $out pack("V", 0x10801000); # Jump Location
print $out pack("V", 0x0); # Reserved
print $out pack("V", 0x0); # DCD pointer
print $out pack("V", 0x0); # Boot Data
print $out pack("V", 0x10e8f000); # Self Pointer
print $out pack("V", 0x10e8f020); # CSF Pointer
print $out pack("V", 0x0); # Reserved
close($out);
x
$ ./genIVT && hexdump ivt.bin
xxxxxxxxxx
0000000 00d1 4120 1000 1080 0000 0000 0000 0000
0000010 0000 0000 f000 10e8 f020 10e8 0000 0000
0000020
  1. Append the IVT file to the end of the padded zImage.
xxxxxxxxxx
$ cat zImage-pad.bin ivt.bin > zImage-pad-ivt.bin

Create the CSF

  1. Create a .csf file that will instruct the CST tool how to sign the linux zImage. Be sure to edit the blocks line to the following format Blocks = <<load_address>> 0x0 <<size_zImage_padded + size_ivt>> before you execute the CST tool.
xxxxxxxxxx
[Header]
Version = 4.0
Hash Algorithm = sha256
Engine Configuration = 0
Certificate Format = X509
Signature Format = CMS
​
[Install SRK]
File = "../../crts/SRK_1_2_3_4_table.bin"
Source index = 0
​
[Install CSFK]
File = "../../crts/CSF1_1_sha256_4096_65537_v3_usr_crt.pem"
​
[Authenticate CSF]
​
[Install Key]
Verification index = 0
Target index = 2
File = "../../crts/IMG1_1_sha256_4096_65537_v3_usr_crt.pem"
​
[Authenticate Data]
Verification index = 2
Blocks = 0x10800000 0x0 0x68f020 "zImage-pad-ivt.bin"
  1. Execute the CST tool to create the HAB data. Next, append the image with the HAB data to create the final signed binary.
xxxxxxxxxx
$ ./cst --o zImage_csf.bin --i zImage.csf
$ cat zImage-pad-ivt.bin zImage_csf.bin > zImage_signed

Booting and Authenticating Linux

Prepare the u-boot environment for booting linux

  1. Specify the type, device number, and partition number of the installation media's fat partition containing the device tree and zImage.
x
=> setenv dev_type usb
=> setenv dev 0
=> setenv dev_part 1
  1. Specify the zImage and device tree load addresses. 
x
=> setenv a_zImage 0x10800000
=> setenv a_fdt 0x13000000
=> setenv ivt 0x0x68f000
=> setenv image zImage
=> setenv image_signed zImage_signed
=> setenv fdt imx6q-sabrelite.dtb
  1. Create a HAB authentication script to verify the kernel. Specify the zImage address and the IVT offset.
xxxxxxxxxx
=> setenv habtest hab_auth_img \${a_zImage} \${ivt}
  1. Create an initialization script that loads the zImage and device tree. Be sure to maintain 4kB alignment. 
xxxxxxxxxx
=> setenv bootinit usb start";" load \${dev_type} \${dev}:\${dev_part} \${a_zImage} \${image}";" load \${dev_type} \${dev}:\${dev_part} \${a_fdt} \${fdt}";" fdt addr \${a_fdt}";" fdt resize 4096
=> setenv bootinitsec setenv image \${image_signed}";" run bootinit";" run fuse_override";" run habtest
  1. Set the default boot script and boot arguments
xxxxxxxxxx
=> setenv bootcmd run bootinitsec";" bootz \${a_zImage} - \${a_fdt}
=> setenv bootargs console=ttymxc1,115200 vmalloc=400M consoleblank=0 rootwait fixrtc board=nitrogen6x root=/dev/sda2
=> savee
  1. Power the device off and then back on again... SECURE BOOT :)
